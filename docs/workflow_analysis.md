## 二、详细工作流程说明

### 1. 爬虫模块工作流程

1. **触发与初始化**
   - 外部 Cron 调用 `crawler/main.py`
   - 支持命令行参数 `--date` 指定爬取日期
   - 默认使用当前日期（仅 07:00-24:00 运行）

2. **文章列表获取**
   - 获取指定日期的文章列表
   - 解析文章元数据（标题、发布单位、链接、发布日期）

3. **增量爬取与去重**
   - 查询数据库获取已存在的文章链接
   - 过滤出新增文章（不在已存在链接集合中）

4. **文章详情获取与附件解析**
   - 获取正文内容
   - 解析附件名称与 URL，仅记录元数据，不下载文件

5. **AI 摘要生成**
   - 对新增文章批量调用 AI 摘要
   - 失败统一重试，最多 3 轮
   - 仍失败写占位摘要并记日志

6. **数据存储与向量**
   - 写入文章表
   - 生成当日向量写入向量表（仅供当日问答）

### 2. 后端 API 工作流程（当前）

1. **认证**
   - 账号密码登录（后端进行校园认证校验）
   - 支持刷新令牌、登出、查询当前用户

2. **文章增量接口**
   - `/articles?date=YYYY-MM-DD&since=ts` 提供增量查询
   - 支持 ETag/Last-Modified 返回 304

3. **AI 问答**
   - `/ai/ask` 使用向量检索，范围由后端配置限制
   - `/ai/embed` 提供文本向量生成

### 3. APP 端工作流程（当前）

1. **鉴权与缓存**
   - 账号/密码登录获取 JWT
   - 当前无自动刷新 token 机制（过期需重新登录）

2. **轮询与通知**
   - 每小时轮询增量接口
   - 发现新文章触发本地通知与角标

3. **阅读状态同步**
   - 已读状态由客户端本地维护（后端不记录）

## 三、旧模块说明
